PROJET · 01 · 18/02/2025
🌐 Installation WordPress sur OVH

Déploiement complet d'un site WordPress sur hébergement OVH : transfert FTP, création de la base de données MySQL, configuration et mise en ligne via le navigateur.

WordPressOVHFileZillaMySQLFTPPHP
+
📅 Réalisé le 18 février 2025

Objectif : Mettre en ligne un site WordPress sur l'hébergement mutualisé OVH, en utilisant un nom de domaine fourni par l'établissement, un client FTP et l'interface d'administration OVH.

01
Téléchargement de WordPress
Récupération de la dernière version depuis wordpress.org. Décompression du dossier en local sur le poste de travail.
02
Connexion FTP et transfert des fichiers
Installation de FileZilla. Connexion au serveur OVH avec les identifiants FTP (hôte, utilisateur, mot de passe). Ouverture du dossier www/ et transfert de l'intégralité des fichiers WordPress vers le serveur distant.
03
Création de la base de données MySQL
Dans l'interface OVH : Web Cloud → Hébergement → Bases de données. Création d'une nouvelle base, définition d'un utilisateur et d'un mot de passe. Note des infos : nom de la BDD, identifiant, mot de passe, serveur SQL.
04
Finalisation via le navigateur
Accès au nom de domaine dans un navigateur. L'assistant WordPress se lance automatiquement. Renseignement des informations de connexion BDD, création du compte administrateur. Site en ligne ✅
Compétences E5 mobilisées : Développer la présence en ligne de l'organisation · Mettre à disposition un service informatique

PROJET · 03
🔒 Partage d'un site web en intranet

Mise en place d'un site web accessible uniquement depuis le réseau local d'une organisation via un serveur Apache/WAMP, sans exposition sur Internet.

ApacheWAMP/XAMPPRéseau localIP privéeDNS localPare-feu
+

Objectif : Rendre un site web accessible uniquement en interne (réseau LAN de l'organisation), sans le publier sur Internet. Utile pour des portails RH, outils internes ou documentation technique d'entreprise.

01
Installation d'un serveur web local
Installation de WAMP (Windows) ou XAMPP sur la machine serveur. Ces solutions embarquent Apache, PHP et MySQL. Démarrage des services Apache et MySQL depuis l'interface.
02
Déploiement des fichiers du site
Copie des fichiers du site dans le dossier www/ (WAMP) ou htdocs/ (XAMPP). Vérification du bon fonctionnement local via http://localhost dans le navigateur.
03
Récupération de l'adresse IP locale
Lancer ipconfig dans l'invite de commandes Windows pour obtenir l'adresse IP privée du serveur (ex : 192.168.1.50). C'est cette adresse que les autres postes utiliseront.
04
Configuration du pare-feu Windows
Autoriser les connexions entrantes sur le port 80 (HTTP) pour Apache dans le pare-feu Windows Defender. Sans cette règle, les autres postes du réseau ne peuvent pas atteindre le serveur.
05
Accès depuis les autres postes du réseau
Les utilisateurs du LAN accèdent au site via http://192.168.1.50 (IP du serveur) dans leur navigateur. Option avancée : modifier le fichier hosts des postes clients pour créer l'adresse http://intranet.local.
